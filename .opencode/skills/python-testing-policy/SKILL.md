# Python Testing Policy

This skill defines the **testing standards and constraints** the AI agent must follow when creating or modifying tests.

All tests must be designed to ensure **reliable, repeatable validation of code behavior** without introducing side effects or interacting with runtime components such as the GUI or application entrypoints.

---

# Core Testing Principles

All tests generated by the agent must be:

- Deterministic
- Non-interactive
- Isolated
- Side-effect safe

Tests must produce **consistent results across repeated runs and environments**.

No test should depend on external system state.

---

# Forbidden Test Behavior

Tests must never perform operations that start or interact with the application runtime.

The agent must ensure that tests do **not**:

- Launch the main application entrypoint
- Launch the GUI
- Launch terminal or CLI interfaces
- Require any user input
- Make real network calls
- Perform destructive filesystem operations

Tests must remain **fully contained within the test environment**.

---

# Allowed Testing Tools

The agent must prefer widely supported and safe testing utilities.

Allowed tools include:

- `pytest`
- `tmp_path` for temporary filesystem usage
- `monkeypatch` for controlled environment modification
- `unittest.mock` for dependency mocking

These tools allow safe isolation of system components during testing.

---

# Test Execution Method

Tests must be designed to run using module-based execution.

The correct command for executing tests is:

```python3 -m pytest
```

The agent must ensure that tests are compatible with this execution method.

---

# Forbidden Test Commands

The agent must never instruct the use of the following command:

```pytest
```

Tests must always be executed using module execution (`python3 -m pytest`) to ensure environment consistency.

---

# Test Isolation Requirements

When interacting with filesystem or environment state, the agent must ensure that tests:

- Use temporary directories (such as `tmp_path`)
- Mock external dependencies
- Avoid modifying real project files
- Avoid modifying system-level environment state

The goal is to ensure that tests remain **fully isolated and safe to run repeatedly**.

---

# When This Skill Applies

This skill applies whenever the agent performs:

- Test generation
- Test modification
- Test refactoring
- Validation of existing tests

All tests introduced into the repository must comply with this policy to maintain **stable and reproducible automated testing**.
