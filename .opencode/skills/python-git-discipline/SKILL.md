# Git Discipline Policy

This skill defines the **Git safety rules and workflow discipline** that the AI agent must follow when interacting with a repository.

The goal is to **protect critical branches, maintain a clean commit history, and prevent destructive Git operations**.

---

# Protected Branches

The following branches are considered **protected production branches**:

- `main`
- `master`
- `production`

These branches represent **stable or production-ready states of the repository** and must not be modified directly by the agent.

---

# Forbidden Actions on Protected Branches

When the repository is currently checked out on a protected branch, the agent must **not perform any direct modifications**.

The agent must never:

- Commit directly to a protected branch
- Modify files while checked out on a protected branch
- Force push
- Reset branch history
- Rebase protected branches
- Delete protected branches

These operations are considered **destructive or high-risk**.

---

# Explicit User Override

The only allowed exception is when the user explicitly provides the instruction:

`Commit directly to main`

Only under this exact instruction may the agent create a commit on the `main` branch.

If this instruction is not present, the agent must **refuse direct commits to protected branches**.

---

# Allowed Branch Naming

When creating new branches, the agent must use one of the following prefixes:

- `feature/<name>`
- `fix/<name>`
- `refactor/<name>`
- `chore/<name>`

---

# Branch Naming Rules

Branch names must follow these conventions:

- Lowercase characters only
- Words separated using hyphens (`-`)
- Concise and descriptive
- Avoid long or ambiguous names

Example:

```
feature/user-authentication
fix/cart-calculation-bug
refactor/payment-service
chore/update-dependencies
```

---

# Commit Message Format

All commits generated by the agent must follow this format:


[AI Generated] <type>: <clear description>


Examples:

```
[AI Generated] feature: add JWT authentication middleware
[AI Generated] fix: correct tax calculation rounding
[AI Generated] refactor: simplify order validation logic
```

Commit messages must remain **clear, concise, and descriptive of the change**.

---

# When This Skill Applies

This skill applies whenever the agent performs operations involving:

- Creating or switching branches
- Modifying repository files
- Generating commits
- Interacting with Git history

The goal is to ensure the repository remains **stable, traceable, and protected from destructive changes**.
